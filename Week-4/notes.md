Week 4: Data Management
Weekly Objectives
Implement local storage using Shared Preferences and SQLite.
Retrieve, update, and display stored records within the application.
Activities Completed
Developed a Record Management module to handle persistent data storage within the application.
Implemented core CRUD (Create, Read, Update, Delete) operations using SQL commands.
Configured a local SQLite database with properly structured tables, fields, and primary keys.
Integrated efficient user interface components such as ListView/RecyclerView to display and manage large sets of data.
Challenges & Solutions
Challenge: Ensuring data persistence across application restarts and maintaining data consistency.
Solution: This was resolved by using Shared Preferences for lightweight session data such as login status, and SQLite for structured and persistent data storage.
Evidence of Work

Database schemas, source code implementations, and record management screenshots will be provided as evidence of completion.
