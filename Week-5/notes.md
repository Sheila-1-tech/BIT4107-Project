Week 5: Networking & APIs
Weekly Objectives
Understand application programming interfaces (APIs), web services, and client-server architecture.
Implement HTTP requests and handle responses asynchronously within a mobile application.
Activities Completed
Enabled the mobile application to communicate with remote servers over the internet.
Consumed RESTful APIs using standard HTTP methods such as GET and POST.
Parsed JSON (JavaScript Object Notation) responses to retrieve and display dynamic data within the application.
Implemented error handling mechanisms to manage network-related issues such as timeouts and server errors (e.g., 404 and 500 status codes).
Challenges & Solutions
Challenge: The application interface became unresponsive when waiting for server responses during network calls.
Solution: This was resolved by implementing asynchronous processing to ensure that network requests run in the background without blocking the user interface.
Evidence of Work

Screenshots of API integration, network request logs, and successful remote data retrieval will be provided as supporting evidence.
