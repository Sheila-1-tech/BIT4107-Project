Week 6 Task 1: Mobile Application Networking Analysis
1. WhatsApp
Client:
The WhatsApp mobile application installed on a user’s smartphone (Android or iOS). The client is responsible for capturing user input such as text messages, voice notes, images, videos, and initiating calls.
Server:
WhatsApp uses Meta-owned cloud-based servers that handle message routing, encryption, delivery, and storage (temporarily for undelivered messages).
Data Exchanged:
Text messages between users
Voice notes and audio data
Images, videos, and documents
Contact information (phone numbers)
Call setup data for voice and video calls
Delivery receipts (sent, delivered, read status)

The communication is highly encrypted using end-to-end encryption, meaning only the sender and receiver can read the messages.

2. Facebook
Client:
The Facebook mobile application or web browser used by users to interact with the platform.
Server:
Meta Facebook servers which manage user accounts, content storage, recommendation systems, and social graph data.
Data Exchanged:
User posts (text, images, videos)
Comments and reactions (likes, emojis)
Profile information (name, email, profile picture)
Friend requests and connections
Messages via Facebook Messenger
Notifications and feed updates

The client sends user actions to the server, which processes and updates data in real time across the platform.

3. Uber
Client:
Uber rider application and Uber driver application installed on smartphones.
Server:
Uber’s cloud-based backend system that manages ride matching, navigation, pricing, and payment processing.
Data Exchanged:
Real-time GPS location of riders and drivers
Ride requests and acceptance status
Trip routes and navigation data
Fare estimates and payment details
Driver availability and ratings
Trip history and receipts

The server continuously processes location data to match riders with nearby drivers efficiently.

4. M-Pesa
Client:
The M-Pesa mobile application or USSD service (*234#) used on mobile phones.
Server:
Safaricom M-Pesa servers integrated with banking systems and financial institutions.
Data Exchanged:
Mobile money transfer requests
Sender and receiver phone numbers
Account balances
Transaction confirmation messages
Payment receipts and transaction IDs
Bill payment and merchant payment data

All transactions are securely processed with authentication to ensure financial safety.

5. YouTube
Client:
YouTube mobile application or web browser used by viewers and content creators.
Server:
Google’s YouTube servers responsible for video storage, streaming, recommendation, and search services.
Data Exchanged:
Video streaming data (audio + video packets)
Search queries
User watch history and preferences
Likes, comments, and subscriptions
Upload data from content creators
Video metadata (titles, descriptions, thumbnails)

YouTube uses adaptive streaming, meaning video quality adjusts based on internet speed.
